# 个人下载器设计规格

日期：2026-07-13

## 目标

开发一个供个人使用的 Windows 便携下载工具。用户粘贴普通 HTTP/HTTPS 文件链接或公开视频页面链接，解析资源后选择保存位置和视频清晰度，完成可暂停、可继续的单任务下载。

交付物是一个便携 ZIP。解压后双击 `app.exe` 即可运行，不要求用户安装 Python。ZIP 内同时包含视频解析及合并所需的独立工具文件。

## 成功标准

第一版同时满足以下条件才算完成：

1. 普通 HTTP/HTTPS 文件能够下载，并能在中断后续传。
2. 至少一个公开测试视频页面能够完成解析、清晰度选择、下载和音视频合并。
3. 下载可以暂停；关闭并重新启动程序后，可以载入未完成任务并继续。
4. 取消任务只删除该任务生成的临时文件，不影响目标目录中的其他文件。
5. 自动化测试全部通过，便携版能够从 `app.exe` 启动。

## 范围

### 第一版包含

- 单窗口、单个活动任务。
- 粘贴或输入 HTTP/HTTPS 链接。
- 普通文件直链下载。
- 公开视频页面解析。
- 视频标题、时长、来源站点和可用清晰度展示。
- 默认最佳画质，也可选择解析结果中存在的具体清晰度。
- HLS/DASH 视频最多 4 路并发分片。
- 下载进度、已下载大小、总大小、速度和预计剩余时间展示。
- 开始、暂停、继续和取消。
- 程序重启后的未完成任务恢复。
- 下载完成后的音视频合并。
- 中文错误信息。

### 第一版不包含

- 浏览器扩展或自动接管浏览器下载。
- 多任务队列和批量下载。
- 播放列表、频道和直播下载。
- 浏览器 Cookie 读取、账号登录或会员内容下载。
- DRM、付费墙或访问控制绕过。
- IDM 式普通 HTTP 文件多分段下载。普通直链仍支持断点续传。
- 自动更新、云同步和远程控制。

## 技术方案

### 技术栈

- Python 3.11 或更新版本。
- Tkinter 桌面界面。
- PyInstaller 生成无控制台窗口的 `app.exe`。
- `yt-dlp` 独立可执行文件负责普通链接下载和公开视频解析。
- `ffmpeg.exe` 与 `ffprobe.exe` 负责音视频合并和媒体探测。
- Python 标准库 `subprocess` 启动外部进程，命令使用参数列表且禁用 shell 解析。
- Python 标准库 `unittest` 编写自动化测试。

使用独立 `yt-dlp` 进程而不是把解析代码冻结进 `app.exe`，原因是解析规则更新频繁，独立引擎更容易替换，并且暂停时可以安全终止进程而不阻塞 GUI。`yt-dlp` 的嵌入、格式选择、进度模板和 FFmpeg 依赖以其[官方文档](https://github.com/yt-dlp/yt-dlp/blob/master/README.md)为准。

### 发布结构

```text
PersonalDownloader/
├── app.exe
├── tools/
│   ├── yt-dlp.exe
│   ├── ffmpeg.exe
│   └── ffprobe.exe
└── LICENSES/
    ├── app-license.txt
    └── third-party-notices.txt
```

不把 FFmpeg 压进单文件 `app.exe`，避免每次启动时解压大型二进制文件。应用代码采用 MIT 许可证；发布包记录所有第三方工具的版本、许可证和来源链接。FFmpeg 的具体许可取决于所选构建，构建脚本必须固定来源并附带对应许可材料。

## 组件边界

### `models.py`

定义任务、解析结果、清晰度选项、进度事件和任务状态。该文件只包含数据结构和状态枚举，不访问文件系统或启动进程。

### `engine.py`

封装 `yt-dlp` 命令构造、资源解析、下载进程、进度行解析以及进程停止。解析使用单条 JSON 输出；下载使用带固定前缀的进度模板，避免依赖面向用户的日志文本。该组件通过事件回调报告进度，不直接操作 GUI。

### `state.py`

负责未完成任务的 JSON 持久化、载入和清除。全局索引位于 `%LOCALAPPDATA%\PersonalDownloader\state.json`，只记录当前单个任务及其专属临时目录。写入时先写临时文件再原子替换，避免程序异常退出造成半写文件。

### `controller.py`

实现任务状态机，协调解析引擎、状态存储和界面事件。它负责验证操作是否合法，例如只有下载中可以暂停、只有已暂停或失败的任务可以继续。该组件不依赖 Tkinter，因此可用假引擎进行自动化测试。

### `ui.py`

构建 Tkinter 单窗口，将用户操作交给控制器，并通过线程安全队列把后台事件送回 GUI 主线程。UI 不直接调用网络或外部工具。

### `main.py`

解析程序目录、定位 `tools`、创建依赖并启动界面。发布后它是 `app.exe` 的入口。

## 用户界面

窗口从上至下包含：

1. 链接输入框，以及“粘贴”“解析”按钮。
2. 解析结果区域，显示标题、时长、站点；普通文件显示文件名和大小。
3. 清晰度下拉框。普通文件不显示该选项；视频默认“最佳画质”。
4. 保存目录和最终文件名输入区域。
5. “开始”“暂停/继续”“取消”按钮。
6. 进度条，以及下载量、速度、剩余时间和当前状态文字。

没有设置页、任务列表和详细日志窗口。按钮是否可用由控制器状态决定。

## 数据流

### 解析

1. 用户输入链接并点击“解析”。
2. 控制器只接受 `http` 或 `https` URL。
3. 引擎运行 `yt-dlp` 的单项目、跳过下载、JSON 输出模式。
4. 引擎把输出转换为解析结果，并过滤出可下载的视频清晰度。
5. 控制器把结果交给 UI，自动填充标题生成的安全文件名。

播放列表 URL 使用单项目模式，不展开整个列表。若 URL 不能对应一个当前视频，则返回不支持错误。

### 下载

1. 控制器确认目标文件不存在、工具齐全且空间路径可写。
2. 在目标目录下创建 `.personal-downloader-tasks/<task-id>/` 专属临时目录。
3. 控制器保存任务状态并启动 `yt-dlp`。
4. 普通文件使用断点续传；HLS/DASH 下载设置最多 4 路并发分片。
5. 视频默认选择最佳视频流和最佳音频流；指定清晰度时选择不高于该高度的最佳组合。
6. FFmpeg 在临时目录内完成合并。
7. 只有引擎成功退出且最终文件存在时，才将文件移动到用户目标路径。
8. 确认目标文件就位后，删除该任务状态和空临时目录。

最终移动发生在同一目标文件系统内，以避免用户看到尚未完成的目标文件。

### 暂停、继续与取消

- 暂停：终止当前引擎进程，等待进程退出，保留 `.part`、分片和任务 JSON，把状态改为“已暂停”。
- 继续：使用完全相同的 URL、格式和输出参数重新启动引擎，由引擎读取临时文件续传。
- 关闭窗口：若正在下载，执行与暂停相同的停止流程，然后退出。
- 取消：先停止进程，再仅删除已记录的任务专属目录及状态索引。删除前必须验证解析后的路径位于 `.personal-downloader-tasks` 下。

## 状态机

允许的主要状态转换如下：

```text
空闲 -> 解析中 -> 可下载 -> 下载中 -> 合并中 -> 完成
                    |          |          |
                    |          +-> 已暂停 +-> 继续 -> 下载中
                    |          +-> 失败   +-> 继续 -> 下载中
                    +---------------------> 已取消 -> 空闲
```

解析失败返回空闲并展示错误。下载失败保留续传文件；完成和取消都会清除当前任务索引。

## 错误处理

- URL 格式错误：解析前拒绝，提示仅支持 HTTP/HTTPS。
- 解析站点不支持：显示引擎归一化后的简短错误，不展示 Python 堆栈。
- 需要登录、受地区限制或疑似 DRM：明确说明第一版不支持，不尝试绕过。
- `yt-dlp` 缺失：解析前提示发布包不完整。
- FFmpeg 缺失：解析仍可进行；需要合并时在下载前拒绝并提示缺少组件。
- 网络中断：使用引擎的有限重试；失败后保留临时文件并允许继续。
- 目标已存在：拒绝覆盖，要求更换名称或目录。
- 状态 JSON 损坏：将其重命名为带时间戳的损坏备份，应用以空闲状态启动。
- 临时目录不安全：取消清理失败并提示用户，不对未验证路径执行递归删除。

## 测试策略

### 自动化测试

- `test_models.py`：状态和值对象的基本约束。
- `test_engine.py`：解析 JSON、清晰度去重、下载参数、进度模板解析和工具缺失错误。
- `test_state.py`：状态保存、原子替换、载入、损坏文件处理和安全清理边界。
- `test_controller.py`：解析、开始、暂停、继续、取消、失败和完成状态转换。
- `test_ui_smoke.py`：窗口可以构造，初始按钮状态正确，不启动真实下载。

外部进程测试使用仓库内的假引擎脚本，模拟解析 JSON、进度事件、暂停及失败。自动化测试不访问真实视频网站。

### 最终验收

1. 启动本地 HTTP 服务，下载已知内容的小文件并校验哈希。
2. 使用一个公开测试视频页面完成解析、清晰度选择、下载和合并。
3. 下载中暂停，关闭应用，重新启动后继续并完成。
4. 取消任务，确认只删除任务临时目录。
5. 构建便携 ZIP，在不依赖系统 Python 的方式下启动 `app.exe`。
6. 运行完整自动化测试套件并确认零失败。

真实网站验收记录测试日期、URL 类型、引擎版本及结果。网站、网络、地区或登录限制导致的失败必须如实报告，不能用自动化假引擎结果替代。

## 安全与使用边界

- 所有外部命令均使用参数数组并关闭 shell 解析，URL 和文件名不会作为命令文本执行。
- 最终文件名经过 Windows 文件名规则清理。
- 应用不读取浏览器 Cookie，不收集或上传用户信息。
- 工具仅用于下载用户有权保存的内容，不提供 DRM 或访问控制绕过能力。
